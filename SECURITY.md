# Security Policy

## Reporting a Vulnerability

Please report security issues privately by opening a GitHub security advisory on the repository, or by contacting the maintainer directly if advisory reporting is not available.

Do not include secrets, printer credentials, Wi-Fi credentials, updater keys, GitHub tokens, or private project files in public issues.

## Scope

Security-sensitive areas include:

- Project import/export
- Firmware or printer communication
- The Orange Pi updater service
- GitHub token handling
- File parsing and generated downloads

## Supported Versions

The `master` branch is the actively supported development line until formal releases are established.
