### B6. Phase A3 spillover (fields originally planned for A3 that need refactoring)
# Moved here from A3 after scoping — each requires touching a subsystem that
# isn't parameterized for the override yet.
[ ] skinOverlapPercent
        → overlap calculation in skin generation is hard-coded to a magic
          number; needs parameterization + careful math so walls don't collide
[ ] extraSkinWallCount
        → skin generation doesn't currently emit extra perimeters around
          skin regions; new loop needed
[ ] topSkinExpandDistance / bottomSkinExpandDistance
        → skin polygons need a clipper-pack inflate; need offset library
[ ] connectInfillLines / connectInfillPolygons
        → generateScanLines / generateLinearInfill return independent
          segments; adding chaining requires nearest-neighbor ordering pass
[x] infillXOffset / infillYOffset
        → wired in generateScanLines (origin shifts). Covers grid/lines/
          triangles/lightning + default patterns; honeycomb/gyroid/cubic
          have their own origin math and aren't yet offset-aware.
[ ] groupOuterWalls
        → wall emission is already per-contour; grouping across contours
          requires buffering all contour walls before emission
[x] innerWallLineWidth
        → wired. Inner-wall extrudes now use innerLW = pp.innerWallLineWidth
          ?? pp.wallLineWidth. Outer wall width unchanged.
[x] supportLineDistance
        → wired. Absolute spacing override in generateSupportForLayer;
          falls back to density-derived spacing when unset.
[x] minimumSupportArea
        → wired. Drops support islands whose merged-overhang bbox area
          is below the threshold (conservative: polygon ≤ bbox area).
[ ] supportHorizontalExpansion / supportJoinDistance
        → still need polygon offset/merge pipeline to wire correctly.
[x] maxCombDistanceNoRetract
        → wired in travelTo. Short travels (below threshold or below
          material.retractionMinTravel) skip retract + Z-hop, approximating
          combing. Real path-aware combing is still TODO.
[ ] avoidPrintedParts / avoidSupports
        → current travel is straight-line. Needs a combing path-planner
          that knows about per-layer printed regions and support regions.
