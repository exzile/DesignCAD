# AI ASSISTANT — MCP server + optional in-app chat  (architectural, multi-session)
# ============================================================================
# Goal: let users drive DesignCAD with their own Claude / Codex / OpenAI
# subscription. Open-source-friendly: no Anthropic API spend on our side,
# no shared key. Two consumption paths:
#   (a) MCP server inside DesignCAD that exposes CAD tools. The user runs
#       Claude Code locally and adds DesignCAD as an MCP server — chat
#       happens in their terminal/IDE, geometry shows up in the running
#       browser session. Subscription quota covers it because Claude Code
#       is the authenticated client.
#   (b) BYOK chat panel inside DesignCAD as a fallback for users who don't
#       run Claude Code. The same MCP tool surface is reused as the
#       function-call schema for the BYOK path so we don't ship two APIs.
#
# Rule of the road: the MCP server is the source of truth. Every CAD
# action the assistant can take must exist as an MCP tool first; the chat
# panel is just a thin client over that surface.
