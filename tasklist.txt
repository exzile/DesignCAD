  CODE AUDIT FINDINGS — deferred from rounds 2-5 (2026-04-18)

--------------------------------------------------------------------------------
ARCHITECTURAL — large refactors, need their own scope
--------------------------------------------------------------------------------
[ ] AUDIT-1. pushUndo() missing from ~50 commit actions in cadStore.ts.
             Currently only called from sketch-editing paths. None of
             commitExtrude/Revolve/Combine/Shell/Draft/Fillet/Chamfer/Hole/
             Thread/Sweep/Loft/Mirror/Pattern*/MeshTransform/Scale/OffsetFace/
             DirectEdit/PlaneCut/Remesh/Fill/Boss/Rib/Web call it. Result:
             undo silently skips ANY feature operation. Best done as a single
             sweep with a `withUndo(action)` helper. (large refactor)

[ ] AUDIT-2. componentStore has NO persist middleware. joints, components,
             bodies, componentConstraints, animationTracks, explodedOffsets,
             rigidGroups, motionLinks all vanish on page refresh. cadStore's
             onRehydrateStorage rebuilds extrude bodies only. Need persist
             middleware + Matrix4 → array serialization. (large refactor)

[ ] AUDIT-3. cadStore partialize omits state fields added in rounds 1-4:
             viewportLayout, ambientOcclusionEnabled, fillBoundaryEdgeData,
             extrudeCreationOccurrence, extrudeTargetBaseFeature,
             constraintSurfacePlane, canvasReferences, jointOrigins, formBodies,
             dimensionToleranceMode/Upper/Lower. All silently reset on reload.

[ ] AUDIT-4. deserializeFeature mesh loss on IDB page-refresh rehydrate.
             Round-3 fix only covers in-memory undo. Mesh-only features
             (D194 fastener, D195 derive, mesh-op imports, tessellate,
             mesh-combine, mesh-smooth, mesh-separate) lose geometry
             permanently on reload. Needs binary mesh persistence (positions
             /index typed arrays in a separate IDB store keyed by feature id)
             or a load-from-source-file replay path. (large)

[ ] AUDIT-5. MultiViewCanvas reparenting collision. The N quadrant Views
             plus the hidden main Canvas all mount <primitive object={
             feature.mesh}>. THREE.Object3D can only have ONE parent — only
             the LAST mount wins, others render empty. Plus PrimitiveBodies/
             ExtrudedBodies/etc. each rebuild geometry N times in 4-quadrant
             mode. Fix needs feature.mesh.clone() per view OR scene-graph
             portals OR a different multi-view strategy. (medium)

[ ] AUDIT-6. JointAnimationPlayer 60Hz set({joints}). componentStore
             tickAnimation runs every frame while playing → full joints
             map replacement → re-renders every body/gizmo/joint origin
             subscribed to joints. Need a render-path that bypasses Zustand
             (mutate Object3D rotations directly in useFrame, or split joints
             into stable-shape vs animated-value stores). (medium)

[ ] AUDIT-7. AnalysisOverlay per-vertex allocations. zebra/draft/curvature
             builders allocate new THREE.Color() and new THREE.Vector3()
             per vertex. Sliding analysis params on a 50k-vertex curvature
             map allocates 50k Color objects per slider tick. Need
             precomputed color LUT + Color reuse via .set(). Plus useMemo
             deps include `bodies` object identity → unrelated body changes
             rebuild everything. (medium)

COMMIT-PATH SAFETY — small validation gaps
--------------------------------------------------------------------------------
[x] AUDIT-8. commitPlaneCut / commitRemesh missing input validation.
             commitPlaneCut: no Number.isFinite(planeOffset) check; NaN
             normal silently produces empty mesh. commitRemesh: no clamp on
             iterations — caller can pass 1000+ and lock the main thread.

[x] AUDIT-9. commitMeshSeparate empty result hides body. When unstitchSurface
             returns 0 geometries, the source feature is filtered out at
             cadStore.ts ~5678 but no replacement exists; user just loses
             the body silently with status "split into 0 parts".

[x] AUDIT-10. commitMeshTransform rollback impossible. Disposes oldMesh
              .geometry immediately after swap. Round-3 stillValid pattern
              would help: defer disposal until snapshot still valid AND
              keep original around for undo.

[x] AUDIT-11. componentConstraint compA existence not validated in
              solveComponentConstraint (only compB checked at line 991).
              Round-5 added stale-pruning in solveAllComponentConstraints
              but the single-constraint path can still write garbage if
              compA was deleted between add and solve.

--------------------------------------------------------------------------------
PARSER / FILE I/O ROBUSTNESS
--------------------------------------------------------------------------------
[x] AUDIT-12. FileImporter.ts 3MF/AMF parsers don't validate triangle
              indices v1/v2/v3 within vertexEls.length. Malformed file
              produces out-of-range setIndex; BufferGeometry accepts but
              every downstream op (computeVertexNormals, raycast, BVH-CSG)
              crashes. (~lines 208-276)

[ ] AUDIT-13. FileImporter.ts createMeshFromPoints (line 363-394). Comment
              says "convex hull" but the loop just buckets every 3
              consecutive STEP CARTESIAN_POINTs into a triangle. STEP
              imports render as random triangle soup. Need a real convex
              hull or proper STEP face/edge parsing.

[ ] AUDIT-14. FileImporter.ts ZIP parser unhandled cases. (a) `<< 24` on
              compSize triggers JS sign-extension; files >= 2 GiB or with
              bit 31 set go negative and bytes.slice returns garbage.
              (b) ZIP general-purpose-bit-3 (size in trailing data
              descriptor — common with streamed 3MF writers) silently skips
              those entries.

[ ] AUDIT-15. JSON.parse without shape validation in cadStore.ts (lines
              ~5408, 5443, 5710 — IDB snapshot + loadFromFile). A truncated
              snapshot whose parse succeeds but is missing `features` array
              crashes inside `.map`.

PERF — LOW priority per-event allocations
--------------------------------------------------------------------------------
[x] AUDIT-16. ExtrudeGizmo.rayToAxisDistance per-drag-frame allocations.
              `new THREE.Ray()` + `ray.origin.clone().sub(...)` per
              pointermove (~60 Hz during drag). Promote scratch Ray +
              Vector3 to module-level.

[ ] AUDIT-17. Viewport.tsx setInterval Quaternion.clone every 100ms while
              camera moves. ~10 Quaternion allocs/sec — minor heap churn.

[ ] AUDIT-18. ConstructionGeometryRenderer / JointOriginRenderer rebuild
              PlaneGeometry / BufferGeometry on every render of every plane
              / origin instance. Cleanup is correct, but singleton-shareable
              geometries are reallocated per instance.

[ ] AUDIT-19. SketchSplineHandles HANDLE_GEO is module-level shared
              SphereGeometry. Not currently a leak (handles aren't features),
              but `tagShared()` only protects MATERIALS — extending to
              geometries would protect any future code path that stored a
              shared geometry on a feature.mesh.
